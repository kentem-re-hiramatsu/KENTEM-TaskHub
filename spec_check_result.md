## 指摘事項

### 🔴 Blocker（仕様と明確に矛盾）

#### [B-1] タブナビゲーション（概要 / タスク / 稼働日管理）が未実装

**実装ファイル**: `features/projects/project-detail/components/project-detail-page-client.tsx`
**仕様根拠**: `docs/spec/screens/0.共通/プロジェクト詳細.md` の「画面構成」

**仕様**:
> 画面は「固定ヘッダー」と「タブコンテンツ」に分かれる。
> ```
> ┌────────────────────────────────────────────┐
> │  固定ヘッダー（常時表示）                   │
> ├────────────────────────────────────────────┤
> │  [ 概要 ] [ タスク ] [ 稼働日管理 ]         │
> └────────────────────────────────────────────┘
> ```

**実装**:
```tsx
// タブはなく、全セクションが縦スクロールで表示される
<VStack className={containerStyle} alignItems="stretch" gap="16px">
  {/* メンバー進捗 Card */}
  {/* タスク一覧 Card */}
  {/* 稼働日設定 Card */}
</VStack>
```

**ズレの内容**:
仕様ではタブで「概要」「タスク」「稼働日管理」の 3 画面を切り替えるが、実装ではタブなしで全コンテンツが縦スクロールで表示される。ユーザー体験・情報構造が大きく異なる。

**修正案**:
ks-react-components の `Tab` コンポーネント（`docs/ksrc/Tab.md` 参照）を使い、3タブを実装する。各タブのコンテンツは以下に対応する：
- **概要**: メンバー一覧 + 進捗推移グラフ
- **タスク**: タスク一覧テーブル
- **稼働日管理**: 稼働日エディタ

### 🟠 Major（仕様の重要部分が未実装/部分実装）

#### [M-1] 固定ヘッダー（進捗カード + コントロール群が常時表示）が未実装

**実装ファイル**: `features/projects/project-detail/components/project-detail-page-client.tsx`
**仕様根拠**: `docs/spec/screens/0.共通/プロジェクト詳細.md` の「固定ヘッダー」

**仕様**:
> 固定ヘッダー（常時表示）: 進捗カード（全体/FE/BE/テスト）/ イテレーション切替 / 進捗集計切替 / 同期ボタン・最終同期時刻

**現状**:
進捗カード・イテレーション切替・集計範囲切替は実装済みだが、`position: sticky` などでヘッダーとして固定されておらず、スクロールで流れてしまう。同期ボタンも未実装（後述 [M-2]）。

**対応案**:
ヘッダー領域に `position: sticky; top: 0` を適用し、タブコンテンツとは別に常時表示されるレイアウトに変更する。

#### [M-2] 同期ボタン（手動データ同期）が未実装

**実装ファイル**: `features/projects/project-detail/components/project-detail-page-client.tsx`
**仕様根拠**: `docs/project_progress_spec.md` §2.1.4「同期フロー（MVP）」/ §6「画面一覧」

**仕様**:
> プロジェクト詳細画面の「同期」ボタンをユーザーが押下して、GitHub API を呼び出し最新のタスク情報を取得する。同期時刻を画面に表示する。

**現状**:
最終同期時刻（`lastSyncedAt`）の表示はあるが、同期ボタンは存在しない。

**対応案**:
固定ヘッダーに「同期」ボタン（ks-react-components の `Button` コンポーネント）を追加する。MVP では UI のみ実装してスタブで可。

#### [M-3] 進捗推移・予想グラフ（折れ線グラフ）が未実装

**実装ファイル**: `features/projects/project-detail/components/project-detail-page-client.tsx`
**仕様根拠**: `docs/spec/screens/0.共通/プロジェクト詳細.md` の「概要タブ > 進捗推移・予想グラフ」

**仕様**:
> 折れ線グラフで以下の 3 本の線を表示する：実績進捗率（実線）/ 理想進捗率（実線）/ 予想進捗率（点線）

**現状**:
モックデータに `trend: TrendPoint[]` が用意されているが、画面には表示されていない。グラフコンポーネント自体が存在しない。

**対応案**:
概要タブに進捗推移グラフコンポーネントを新規作成する。グラフライブラリは Recharts 等を使用する方向で確認が必要。`trend` データの `actualRate`・`idealRate`・`predictedRate` を 3 本の折れ線で表示する。

#### [M-4] 稼働日管理の「変更メモ」列・変更時赤字表示が未実装

**実装ファイル**: `features/projects/project-detail/components/working-days-editor.tsx`
**仕様根拠**: `docs/project_progress_spec.md` §2.5.2「稼働日の個別変更」

**仕様**:
> | 変更フラグ | デフォルトから変更された場合、**値を赤字で表示** |
> | 変更メモ | 変更理由などを自由記述（例：「有給3日取得」）|

**現状**:
稼働日数の入力のみ実装済み。変更フラグ（赤字表示）・変更メモ入力欄がない。

**対応案**:
- デフォルト稼働日数（例：10日）をシステム定数として定義し、変更された場合は稼働日数入力欄のテキストを赤色で表示する
- 各行に変更メモのテキスト入力欄（`TextInput`）を追加する
- データ型 `WorkingDaysState` に `memo: string` と `isModified: boolean` を追加する
