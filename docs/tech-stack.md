# 技術選定

## フロントエンド

### Next.js

| 項目 | 内容 |
|------|------|
| バージョン | App Router（v14 以降） |
| 言語 | TypeScript |

**採用理由**

- React ベースの SSR / RSC により、初期表示パフォーマンスと SEO を両立できる
- App Router によるファイルベースルーティングで、画面追加・分割が容易
- Vercel エコシステムとの親和性が高く、デプロイ・プレビュー環境の構築が簡単

## バックエンド

### Hono

| 項目 | 内容 |
|------|------|
| ランタイム | Node.js（または Edge Runtime） |
| 言語 | TypeScript |

**採用理由**

- 名前がかっこよかったから
- 軽量かつ高速な Web フレームワークで、Edge / Cloudflare Workers にも対応
- Express に近い API 設計で学習コストが低い
- TypeScript ファースト設計により、リクエスト・レスポンスの型安全性を確保しやすい
- RPC モード（`hono/client`）を使うことで、FE/BE 間の型共有が可能

### Prisma

| 項目 | 内容 |
|------|------|
| 用途 | ORM（データベースアクセス） |
| 言語 | TypeScript |

**採用理由**

- スキーマファースト設計により、DB 定義とアプリコードを一元管理できる
- 自動生成される型安全なクライアントで、クエリのミスをコンパイル時に検出できる
- マイグレーション管理機能（`prisma migrate`）により、スキーマ変更の追跡・適用が容易
