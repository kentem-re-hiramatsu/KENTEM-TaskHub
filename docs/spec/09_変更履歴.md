## 9. 変更履歴

| バージョン | 日付 | 変更内容 |
|-----------|------|---------|
| 1.0.0 | 2026-04-01 | 初版作成 |
| 1.1.0 | 2026-04-01 | 完了ステータス定義を PL 手入力方式に変更・運用フロー追加・データモデル更新 |
| 1.3.0 | 2026-04-01 | カラムマッピング項目を更新（予定時間・実施時間・ポイント・区分追加）・開発効率指標を追加 |
| 1.4.0 | 2026-04-01 | 予想進捗グラフ機能を追加（折れ線グラフ・直近Nイテレーション平均ベロシティ・N数ユーザー変更可）|
| 1.5.0 | 2026-04-01 | 区分にテストを追加・FE/BE/テストの総工数集計機能を追加（プロジェクト全体・イテレーションごと）|
| 1.6.0 | 2026-05-09 | **GitHub Projects API 連携をメイン機能化**（§2.1 新設）。CSV インポートをサブ機能（GitHub未利用 / Backlog 利用者向け）に降格（§2.2）。データソース種別を `data_source_type` で管理。データモデルに `ApiConnection` / `SyncSnapshot` を追加し、`ColumnMapping` を `FieldMapping` に汎化（API/CSV 両対応）。認証方式・API 同期失敗時の挙動を §8 に追加 |
| 1.7.0 | 2026-05-09 | **イテレーションカレンダー方式に変更**（§2.4 改訂）。プロジェクト個別の `iteration_start_date` 自動算出を廃止し、管理者が年度単位で登録する全社共通カレンダー（変則期間対応）から取得する方式に変更。プロジェクト設定に「参照カレンダー」「開始イテレーション」を追加。データモデルに `IterationCalendar` / `IterationCalendarEntry` を追加し、`Project.Iterations[]` を `ProjectIterationStates[]`（カレンダーエントリ参照型）に再構成。画面一覧に「イテレーションカレンダー管理」を追加 |
| 1.8.0 | 2026-05-13 | **認証仕様を確定**。ログイン方式を GitHub OAuth に固定し、`@kentem` 所属以外のアカウントをアクセス拒否する要件を追加。関連して §2.1.1 と §5 を更新し、認証関連の TBD を解消 |
| 1.9.0 | 2026-05-13 | **プロジェクト詳細の表示要件を拡張**。メンバー一覧に計画ベロシティ（目標ベロシティ）と開発効率を追加し、イテレーションごとの稼働日合計を確認できる仕様を追加。メンバー一覧のイテレーション切替（過去/現在のみ選択可、未来は選択不可）を追加。開発効率の定義（実施時間合計 ÷ 予定時間合計）を明記 |
| 1.10.0 | 2026-05-14 | **プロジェクト詳細の進捗カード集計範囲切替を追加**。イテレーション切替と独立して `選択のみ` / `選択まで累計` を選べる要件を追加し、各モードの表示定義を明記 |
| 1.11.0 | 2026-05-14 | **設定画面を拡張**。サイドメニュー名称を「設定」に変更し、設定配下に「権限」と「メンバー登録」を定義。メンバー登録は GitHub ログインに依存しない手動登録（表示名必須、Login任意）を許容 |
| 1.12.0 | 2026-05-14 | **権限モデルを簡素化**。ユーザー権限を「管理者 / 標準」の2区分に統一。メンバー登録は GitHub login 必須とし、初回ログイン時に login 一致で自動紐づけする仕様を追記 |
| 1.13.0 | 2026-05-14 | **設定画面に「イテレーションカレンダー設定」を追加**。アクセス経路を `設定 > イテレーションカレンダー設定` とし、操作権限を管理者のみに制限する仕様を追記 |
| 1.14.0 | 2026-05-15 | **権限モデルを3区分に拡張**。ユーザー権限を「開発者(superAdmin) / 管理者(admin) / 標準(user)」に変更し、上から権限が強い順序を追記。管理者専用機能の操作権限を「開発者・管理者」に更新 |
| 1.15.0 | 2026-05-15 | **閲覧権限と2nd以降対応を追記**。`user` は自分の過去情報のみ閲覧可、`admin` / `superAdmin` は各ユーザーの過去情報を閲覧可とする要件を追加。`user` の未所属プロジェクト詳細を非公開（遷移不可）とする制御を追加。2nd 以降として見積もり基準対応・ベロシティ算出導入を追記 |
| 1.16.0 | 2026-05-15 | **個人設定を追加**。`設定 > 個人設定` の項目を `GitHub Account Name / 表示名 / GitHub Account ID / 権限` とし、編集可能項目を「表示名のみ」に制限する仕様を追記 |
| 1.17.0 | 2026-05-15 | **個人設定URLを単体化**。個人設定のアクセスURLを `/settings/personal` から `/personal-settings` に変更し、設定配下ではない独立ルートに移行 |
| 1.18.0 | 2026-05-15 | **画面仕様を分割管理に変更**。`06_画面一覧` を各画面仕様へのリンク集に変更し、`docs/spec/screens/` 配下へ画面ごとの仕様ファイルを新設 |
