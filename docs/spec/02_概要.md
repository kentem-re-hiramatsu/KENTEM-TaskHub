## 1. 概要

### 1.1 背景・目的

#### 現状の課題

現在プロジェクトの進捗管理は Excel で行っているが、以下の課題がある。

- **更新が手間**：PL が手動で Excel を更新する必要があり、更新負荷が高い
- **リアルタイム性がない**：PL が更新しない限り他のメンバーは最新の進捗を確認できない
- **進捗把握がPL依存**：メンバー自身が進捗を確認しづらく、各自の当事者意識が生まれにくい
- **横断管理ができない**：上位役職者が複数プロジェクトの進捗を一覧で確認する手段がない

#### 目的

- **GitHub Projects API との連携によるリアルタイム進捗反映**を実現し、PL の更新負荷を最小化する
- **リアルタイムで進捗を共有**することで、PL 以外のメンバーも常に最新の進捗を把握できるようにする
- メンバー各自が進捗を可視化することで、**それぞれの責任感・当事者意識を高める**
- 上位役職者が**複数プロジェクトの進捗を横断して確認**できる環境を整える
- GitHub Projects を利用していないプロジェクト（Backlog 等）には**CSV インポートをサブ機能として提供**する

### 1.4 今後の展望

MVP 以降の段階で以下の拡張を想定している。

| # | 展望 | 説明 |
|---|------|------|
| 1 | **API 自動同期（Webhook / スケジュール同期）** | GitHub Webhook やスケジュール実行で API 同期を自動化し、手動更新ボタン操作なしでも進捗が反映されるようにする |
| 2 | **Backlog API 連携** | Backlog の API にも対応し、Backlog 利用プロジェクトでも CSV インポート不要にする |
| 3 | **全社への展開** | 全プロジェクトが Excel での進捗管理を廃止し、本 Web アプリに一本化する |
| 4 | **見積レビュー会での活用** | 見積レビュー会の場でも本アプリの進捗・ベロシティデータを参照・活用できるようにする |
