# プロジェクト設定

## 概要

個別プロジェクトの基本情報・マッピング設定を変更する。

**アクセス経路：** プロジェクト詳細ヘッダーの「設定」リンク

---

## 権限

| 操作 | 必要権限 |
|------|---------|
| 設定画面へのアクセス | `admin` / `superAdmin` |
| 設定の保存 | `admin` / `superAdmin` |

---

## 画面構成

```
┌────────────────────────────────────────────┐
│  ← 詳細に戻る                              │
│  <プロジェクト名> — 設定                   │
├────────────────────────────────────────────┤
│  [基本情報]                                │
│  [フィールドマッピング]                    │
│  [ステータス・区分マッピング]              │
├────────────────────────────────────────────┤
│                              [ 保存 ] ← 固定│
└────────────────────────────────────────────┘
```

---

## 基本情報セクション

プロジェクト登録時に設定した基本情報を変更できる。

| 項目 | 説明 |
|------|------|
| プロジェクト名 | 任意の名称 |
| バージョン | バージョン番号 |
| 期間 | 開始日〜終了日 |
| Project URL | GitHub Projects の URL（GitHub Projects API 利用時）|
| Project 番号 | GitHub Projects のプロジェクト番号（GitHub Projects API 利用時）|

---

## フィールドマッピングセクション

外部ツール（GitHub Projects / CSV）の列名とアプリ内項目のマッピングを変更できる。

| アプリ内項目 | 説明 |
|-------------|------|
| 予定時間 | 取得データの対応列を指定 |
| 実施時間 | 取得データの対応列を指定 |
| ポイント | 取得データの対応列を指定 |
| ステータス | 取得データの対応列を指定 |
| 担当者 | 取得データの対応列を指定 |
| イテレーション | 取得データの対応列を指定 |

登録時と異なり、列名はテキスト入力で直接指定する（ドロップダウン選択なし）。

---

## ステータス・区分マッピングセクション

### ステータスマッピング

システム側ステータスごとに、外部ツールのステータス値を複数登録できる。

| システム側ステータス | 説明 |
|------------------|------|
| 未対応 | 未着手・バックログとみなす外部ステータス値を指定 |
| 実装中 | 実装中とみなす外部ステータス値を指定 |
| プルリク中 | レビュー中とみなす外部ステータス値を指定 |
| 完了 | 完了とみなす外部ステータス値を指定 |
| 破棄 | 破棄・クローズとみなす外部ステータス値を指定 |

### 区分値マッピング

システム側区分ごとに外部ツールのラベル値を複数登録できる。

| システム側区分 | 説明 |
|------------|------|
| FE | フロントエンドとみなすラベル値を指定 |
| BE | バックエンドとみなすラベル値を指定 |
| テスト | テストとみなすラベル値を指定 |

---

## 保存ボタン

- 画面右下に固定表示する
- クリックすると入力内容をローカルステートに反映し、「保存しました」と 2 秒間表示する
- ページリロードで設定は元に戻る（API 永続化は将来対応）

---

## 戻るリンク

ヘッダーの「← 詳細に戻る」をクリックすると `/projects/[id]` へ遷移する。
