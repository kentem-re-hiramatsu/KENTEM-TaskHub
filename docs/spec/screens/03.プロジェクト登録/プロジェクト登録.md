# プロジェクト登録

## 概要

プロジェクトの新規作成・編集を行う。5ステップのウィザード形式で、基本情報・メンバー設定・稼働日・マッピングを順に設定する。

**アクセス経路：** サイドバー「プロジェクト登録」

---

## 権限

| 操作 | 必要権限 |
|------|---------|
| 新規作成・編集・削除 | `admin` / `superAdmin` のみ |
| サイドバーへの表示 | `admin` / `superAdmin` のみ |

---

## ステップ構成

### ① プロジェクト基本情報

| 項目 | 説明 |
|------|------|
| プロジェクト名 | 任意の名称 |
| データソース種別 | **GitHub Projects API** / **CSV** から選択 |
| 期間 | 開始日〜終了日 |

#### GitHub Projects API を選択した場合

| 項目 | 説明 |
|------|------|
| Project URL | GitHub Projects の URL |
| Project 番号 | GitHub Projects のプロジェクト番号 |
| API 確認ボタン | 入力した接続情報で API アクセスを検証する |

#### CSV を選択した場合

| 項目 | 説明 |
|------|------|
| ファイル取込み | CSV ファイルをアップロードする |

#### 次へボタン活性化条件

以下をすべて満たした場合に「次へ」ボタンが活性化される。

| 条件 | GitHub Projects API | CSV |
|------|---------------------|-----|
| 必須項目がすべて入力されている | プロジェクト名・期間・Project URL・Project 番号 | プロジェクト名・期間・ファイル |
| データソースに問題がない | API 確認ボタンで接続成功 | アップロードした CSV にエラーがない |

---

### ② メンバー設定

プロジェクトに参加するメンバーを追加・設定する。複数メンバーの登録が可能。

| 項目 | 説明 |
|------|------|
| 名前 | セレクターから選択する |
| 区分 | メンバーの担当区分（例：FE / BE / テスト）|
| 計画ベロシティー | メンバーの1イテレーション当たりの計画ポイント |

---

### ③ 稼働日設定

メンバーごとに各イテレーションの稼働日を設定する。

| 項目 | 説明 |
|------|------|
| メンバー | ②で登録したメンバー一覧 |
| 各イテレーションの稼働日 | 開発者のイテレーションカレンダーを元に稼働日が初期値として入力される |

---

### ④ マッピング設定

API または CSV から取得した列と、アプリ内項目をマッピングする。

| アプリ内項目 | 説明 |
|-------------|------|
| 予定時間 | 取得データの対応列を指定 |
| 実施時間 | 取得データの対応列を指定 |
| ポイント | 取得データの対応列を指定 |
| ステータス | 取得データの対応列を指定 |
| 担当者 | 取得データの対応列を指定 |
| イテレーション | 取得データの対応列を指定 |

---

### ⑤ ステータス・ラベルマッピング

取得データのステータス値・ラベル値とアプリ内の区分をマッピングする。

#### ステータスマッピング

システム側ステータス（固定）ごとに、外部ツール（GitHub Projects 等）のステータス値を Enter で複数登録できる。

| システム側ステータス | 説明 |
|------------------|------|
| 未対応 | 未着手・バックログとみなす外部ステータス値を指定 |
| 実装中 | 実装中とみなす外部ステータス値を指定 |
| プルリク中 | レビュー中とみなす外部ステータス値を指定 |
| 完了 | 完了とみなす外部ステータス値を指定 |
| 破棄 | 破棄・クローズとみなす外部ステータス値を指定 |

#### 区分値マッピング

システム側区分ごとに外部ツール（GitHub Labels）のラベル値を Enter で複数登録できる。

| システム側区分 | 説明 |
|------------|------|
| FE | フロントエンドとみなすラベル値を指定 |
| BE | バックエンドとみなすラベル値を指定 |
| テスト | テストとみなすラベル値を指定 |
