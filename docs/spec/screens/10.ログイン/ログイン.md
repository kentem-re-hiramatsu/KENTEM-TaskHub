# ログイン

## 概要

GitHub OAuth を使ってアプリケーションにログインする。

**アクセス経路：** 未ログイン状態でのアクセス（`/login`）、または認証切れによるリダイレクト

---

## 認証方式

| 方式 | 状態 |
|------|------|
| GitHub OAuth | サポート |
| ゲストログイン | 要検討 |

---

## 初回ログインフロー

GitHub OAuth 認証完了後、初めてアプリを利用するユーザーに対して以下のフローを行う。

1. GitHub から取得した情報を自動入力した確認画面を表示する
2. ユーザーが内容を確認・編集して「登録」ボタンを押すとアカウントを登録する
3. 登録完了後、担当プロジェクト画面へリダイレクトする

### 確認画面の入力項目

| 項目 | 入力方式 | 初期値 | 備考 |
|------|---------|--------|------|
| GitHub Account ID | 自動入力・変更不可 | GitHub から取得 | |
| GitHub Account Name | 自動入力・変更不可 | GitHub から取得 | |
| 表示名 | 手入力可 | GitHub Account Name と同じ | アプリ内表示用の名前 |
| 権限 | 非表示 | `user`（標準） | 登録時に自動付与 |

---

## 2回目以降のログインフロー

1. GitHub OAuth 認証完了
2. 担当プロジェクト画面へリダイレクト

---

## ログイン後遷移先

- 担当プロジェクト画面（`/`）

---

## OAuth スコープ不足時の再同意フロー

詳細は [`10_OAuth再同意フロー.md`](../../10_OAuth再同意フロー.md) を参照。

### 基本方針

- ログイン成功 ≠ Projects API 利用可能
- ログイン後、Projects メタ情報取得 API を実行してスコープを確認する
- スコープ不足の場合、専用エラー UI を表示して再認可を促す

### 必須スコープ

| 用途 | スコープ |
|------|---------|
| 参照のみ | `read:project` |
| 更新を含む | `project` |

### エラー UI

- 不足スコープ名を表示する
- 再認可ボタン（再ログイン / 再同意）を提供する
- 再認可完了後に同じ API を再試行する

**メッセージ例：**
> 「GitHub Projects の参照権限が不足しています。`read:project` 権限で再度連携してください。」

---

## セキュリティ要件

[`10_OAuth再同意フロー.md` §10.6](../../10_OAuth再同意フロー.md) のトークン管理方針に準拠する。

- ユーザーの OAuth ログイン後トークンをサーバー側で一時利用する（ブラウザへ返却しない）
- セッションは `httpOnly` Cookie 等で管理し、フロント JS から直接参照できない形にする
- DB への長期保存を避け、短寿命トークンを優先する
- スコープは最小権限（参照のみなら `read:project`）に限定する
- CSRF 対策を実装する
- アプリログにトークン文字列を出力しない（マスクまたは非出力）

---

## ログ要件

スコープ不足発生時に以下を記録する。

| 記録項目 | 内容 |
|---------|------|
| login | user / organization |
| project number | プロジェクト番号 |
| 発生日時 | タイムスタンプ |
| GraphQL error type | `INSUFFICIENT_SCOPES` |
| 不足スコープ | 不足しているスコープ名 |

---

## 権限

- 未ログインユーザーのみアクセス可能
- ログイン済みユーザーがアクセスした場合は担当プロジェクト画面へリダイレクト
