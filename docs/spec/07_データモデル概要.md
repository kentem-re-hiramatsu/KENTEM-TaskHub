## 7. データモデル（概要）

```
SystemConfig                             ← 会社全体の固定設定
  └── default_working_days_per_iteration ← デフォルト稼働日数（システム管理者が設定）

IterationCalendar                        ← 全社共通・年度ごとのイテレーションマスタ（管理者が登録）
  ├── id
  ├── fiscal_year                        ← 年度（例：2026）
  ├── name                               ← 任意名（例：「2026年度カレンダー」）
  └── Entries[]                          ← 各イテレーションの期間（変則期間も可）
        ├── id
        ├── number                       ← イテレーション番号（1, 2, 3, ...）
        ├── start_date / end_date        ← 開始日・終了日（管理者が登録、変則対応）
        └── memo                         ← 変則理由などの自由記述

Project
  ├── id
  ├── name
  ├── data_source_type                   ← api_github / csv
  ├── iteration_calendar_id              ← 参照する全社カレンダー
  ├── start_entry_id                     ← カレンダー上の開始イテレーション
  ├── ApiConnection                      ← data_source_type = api_github の場合のみ
  │     ├── project_url                  ← GitHub Projects URL
  │     ├── owner / repo
  │     ├── project_number
  │     └── auth_credential_ref          ← 認証情報への参照（保管方法は §8 TBD）
  ├── FieldMapping                       ← マッピング設定（プロジェクト単位）。API/CSV 両対応
  │     ├── source_kind (api / csv)
  │     ├── title_source                 ← API: 固定 / CSV: カラム名
  │     ├── status_source
  │     ├── assignee_source
  │     ├── iteration_source
  │     ├── estimated_hours_source
  │     ├── actual_hours_source
  │     ├── points_source                ← API: フィールド名 or Labels(`sp:N`) / CSV: カラム名
  │     ├── division_source_kind         ← labels / custom_field / csv_column
  │     ├── division_source_name         ← フィールド名 or カラム名（Labels の場合は無視）
  │     ├── done_statuses[]              ← 「完了」とみなす Status 値リスト
  │     ├── in_review_statuses[]         ← 「プルリク中」とみなす Status 値リスト
  │     ├── include_in_review_in_progress ← プルリク中を進捗に含めるか
  │     ├── fe_values[]                  ← FE とみなす値リスト
  │     ├── be_values[]                  ← BE とみなす値リスト
  │     └── test_values[]                ← テストとみなす値リスト
  ├── Members[]                          ← プロジェクト全体のメンバーマスタ
  │     ├── name
  │     ├── division (FE / BE / テスト)
  │     └── planned_velocity             ← 計画ベロシティ（PL が設定）
  ├── TotalPoints                        ← 同期時に自動集計・更新
  │     ├── fe_total
  │     ├── be_total
  │     └── test_total
  └── ProjectIterationStates[]           ← カレンダーエントリごとのプロジェクト状態
        ├── iteration_calendar_entry_id  ← 参照するカレンダーエントリ
        ├── fe_points_total              ← そのイテレーションの FE ポイント合計
        ├── be_points_total              ← そのイテレーションの BE ポイント合計
        ├── test_points_total            ← そのイテレーションのテストポイント合計
        ├── IterationMembers[]           ← イテレーションごとの稼働情報
        │     ├── member_id
        │     ├── working_days
        │     ├── is_working_days_modified
        │     ├── working_days_memo
        │     └── target_points
        └── SyncSnapshots[]              ← API 同期 / CSV インポートの実行ごとに保存
              ├── synced_at              ← 同期実行時刻
              ├── source_kind (api / csv)
              └── Tasks[]                ← この同期時点でのタスク一覧
                    ├── external_id      ← Issue 番号 / 課題キーなど
                    ├── title
                    ├── status
                    ├── points           ← 計画ポイント
                    ├── estimated_hours
                    ├── actual_hours
                    ├── assignee
                    └── division (FE / BE / テスト / unclassified)
```

> 💡 **イテレーションエンティティの統合**：従来 `Project.Iterations[]` として持っていた「番号・開始日・終了日」は `IterationCalendar.Entries[]` に統合し、Project 側はカレンダーエントリへの参照（`ProjectIterationStates[]`）で集計値・稼働情報・スナップショットを保持する。

> 💡 **タスクはスナップショット単位で保存**：同じイテレーション内で複数回同期した場合、最新スナップショットを進捗計算に使用しつつ、過去スナップショットも履歴として参照できる。
