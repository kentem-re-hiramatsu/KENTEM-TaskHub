## 未決事項

| # | 項目 | 内容 | 優先度 |
|---|------|------|--------|
| 1 | ホスティング環境 | クラウド / オンプレ / ローカル | 高 |
| 2 | ストーリーポイントの算出方法 | GitHub Projects のどのフィールド形式（カスタム数値 / Label `sp:N` 等）まで対応するか | 中 |
| 3 | API 同期失敗時の挙動 | レート制限 / 認証切れ / ネットワーク不通時のリトライ・通知方針 | 中 |
| 4 | データのバックアップ・エクスポート | DB ダンプ or CSV 出力機能の要否 | 低 |
| 5 | ユーザー別履歴画面の導線 | 「ユーザー一覧から遷移」か「サイドメニューから選択」かを確定する | 中 |
| 6 | 2nd 以降の見積もり基準対応 | 見積もり基準（ポイント/工数の換算ルール）をどこまで設定可能にするかを定義する | 中 |
| 7 | 2nd 以降のベロシティ算出 | 算出ロジック・表示単位・比較軸（個人/チーム/プロジェクト）を定義する | 中 |
| 8 | 見積基準UIの前提 | 単一登録前提UI（1件固定表示）か複数登録前提UI（一覧+切替）かを検討する | 中 |
| 9 | PWA対応の要否 | PWA（ホーム画面追加、オフライン対応、プッシュ通知）を提供するか | 中 |
| 10 | URLの検討 | 画面・機能ごとのURL設計（命名規則、階層構造、クエリパラメータ方針）を定義する | 中 |
